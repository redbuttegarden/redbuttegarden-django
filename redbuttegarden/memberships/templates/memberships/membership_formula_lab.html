Membership formula preview

Edit the ordered formulas and price fallback rules for each recommendation slot, then preview the live recommendation engine against the current active membership levels or download a fixture-backed matrix workbook that matches the command output.

This page has two kinds of fields. The four multi-line boxes under Recommendation formulas describe the exact membership setup to try for each slot. The four single-line boxes under Price fallback formulas are backup rules that are only used when the matching box above does not return a result.

In the Recommendation formulas boxes, each line is checked from top to bottom, and the first line that finds a matching membership is used. These boxes use short labels: C for cardholders, G for guests, and T for tickets. prev(T) means the next lower ticket step and next(T) means the next higher ticket step. Example: (C, G, prev(T)).

In each field under Price fallback formulas, enter one backup rule such as cheaper(1) or expensive(2). Here, 1 means “use the first matching backup option,” 2 means “use the second matching backup option,” and so on.

By default, a backup rule first looks for memberships with the same number of cardholders as the request, then widens to the closest remaining prices if needed. You can narrow or guide that search by adding extra options inside the same rule.

match=... limits the search to memberships that match the requested values you name. For example, cheaper(1; match=cardholders,guests) only considers memberships with the same cardholder count and guest count as the request.

prefer=... changes what the system tries to keep the same before widening the search. For example, expensive(2; prefer=guests) tries to keep the same guest count first, even if the cardholder count changes. Use prefer=any if you do not want any “keep this the same first” step.

The values you can use with match=... and prefer=... are cardholders, guests, and tickets. Short forms C, G, and T also work. Examples you can paste directly into a price fallback field: cheaper(1; match=cardholders), cheaper(1; match=cardholders,guests), expensive(1; prefer=tickets), cheaper(1; prefer=any; match=tickets).
